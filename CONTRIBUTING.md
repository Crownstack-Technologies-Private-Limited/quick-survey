# Contributing to Quick Survey 📊

First off, thank you for considering contributing to Quick Survey! It's people like you that make Quick Survey such a great tool.

## 🌈 How Can I Contribute?

### Reporting Bugs

- **Check if the bug has already been reported** by searching on GitHub under [Issues](https://github.com/Crownstack-Technologies-Private-Limited/quick-survey/issues).
- If you can't find an open issue addressing the problem, [open a new one](https://github.com/Crownstack-Technologies-Private-Limited/quick-survey/issues/new). Be sure to include a **title and clear description**, as much relevant information as possible, and a **code sample** or an **executable test case** demonstrating the expected behavior that is not occurring.

### Suggesting Enhancements

- **Check if the enhancement has already been suggested** by searching on GitHub under [Issues](https://github.com/Crownstack-Technologies-Private-Limited/quick-survey/issues).
- [Open a new issue](https://github.com/Crownstack-Technologies-Private-Limited/quick-survey/issues/new) and describe the enhancement you'd like to see, why it's useful, and how it might be implemented.

### Your First Code Contribution

Unsure where to begin contributing? You can start by looking through these `good-first-issue` and `help-wanted` issues:

- [Good First Issues](https://github.com/Crownstack-Technologies-Private-Limited/quick-survey/labels/good-first-issue) - issues which should only require a few lines of code, and a test or two.
- [Help Wanted Issues](https://github.com/Crownstack-Technologies-Private-Limited/quick-survey/labels/help-wanted) - issues which should be a bit more involved than `good-first-issue` issues.

### Pull Requests

1. **Fork the repo** and create your branch from `main`.
2. **Install dependencies** (`bundle install` and `yarn install`).
3. **Write your code** and ensure it follows the project's coding standards.
4. **Write tests** for your changes.
5. **Run the test suite** (`bin/rails test`) to make sure everything is working.
6. **Submit a pull request** with a clear description of your changes.

## 📝 Coding Standards

- We use **RuboCop** to enforce a consistent coding style. Run `bundle exec rubocop` before submitting your PR.
- Follow standard Rails conventions.
- Keep methods short and focused.
- Comment complex logic where necessary.

## 🧪 Testing

We use **Minitest** for our testing suite. Please ensure that:
- All existing tests pass.
- New features or bug fixes include corresponding tests.

## 🤝 Community

If you have questions or want to discuss ideas, feel free to join our [GitHub Discussions](https://github.com/Crownstack-Technologies-Private-Limited/quick-survey/discussions).
